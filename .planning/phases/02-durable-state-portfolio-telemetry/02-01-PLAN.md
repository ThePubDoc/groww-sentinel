## Phase Goal

**As** the portfolio owner, **I want** the morning digest to remember each stock's
price peak and my portfolio value across runs and to flag corporate-action-distorted
cost, **so that** trailing-stops and averaging calls fire on real history instead of
resetting every run, and a bonus/split never triggers a false STOP or BOOK.

*(Derived from the PROJECT.md / CLAUDE.md Core Value; the ROADMAP Phase 2 Goal is
outcome-shaped, not in As/I-want/so-that form. This slice delivers the corp-action-safety
half of that goal in the pure core + presentation layers; plan 02-02 makes it durable.)*

<objective>
Extend the pure rules core so a corporate action (bonus/split) can no longer masquerade
as a real STOP/BOOK/AVERAGE, and render that new CORP ACTION verdict in the digest.
This is the pure + presentation half of RULES-06 and the durable-peak mechanics
(STATE-01/02/03) — no I/O, boundary-testable first.

Per D-09: detect a distorted average cost via a quantity jump vs stored prior qty
without a proportional cost-basis change, and replace the P&L-based flag with a
CORP ACTION warning (suppress pct). Per D-11: rescale the stored peak by prior_qty/qty
so TRAIL WATCH keeps evaluating on a comparable number. Per D-04: new_state is rebuilt
from current holdings each run (rebuild-not-merge), now carrying qty/avg_cost forward as
the corp-action baseline.

Purpose: a distorted cost basis is a money-path bug — one bad bonus adjustment silently
mis-flags every affected symbol. This closes it deterministically in the testable core.
Output: rules.py corp-action logic + peak rescale + extended new_state shape; notify.py
CORP ACTION rendering; extended unit tests. No live state yet (sentinel still passes {})
— 02-02 wires persistence that makes this fire end-to-end.
