## Phase Goal

**As** the portfolio owner, **I want** the digest header to carry P&L, day change, an
N-day trend, and an intraday %, **so that** I see portfolio direction at a glance without
opening the app. This slice adds the telemetry that stored snapshots (02-02) make possible.

<objective>
Report portfolio telemetry in the digest: overall unrealized P&L (PNL-01, already computed
in Phase 1 — surface it), day change vs the prior day's stored snapshot (PNL-02), an N-day
trend from stored snapshots (PNL-03), and an intraday % when the price source exposes
previous close (PNL-04).

Per D-07 the header shows a compact telemetry line (e.g. `P&L +17.1% · Day +1.2% · 5d ↗
+3.4%`; exact glyphs/wording and header-vs-section placement are Claude's discretion per
D-07/02-CONTEXT). Per D-12 the day-change baseline is the most recent snapshot strictly
before today — NOT sorted(keys)[-2], which breaks on the 2nd+ hourly run of a day. Trend
degrades gracefully across the 0 / 1-4 / 5+ prior-entry cases and labels with the actual
window length (02-RESEARCH Pattern 4). Intraday % uses yfinance fast_info.previous_close /
last_price (02-RESEARCH), worded defensively per Pitfall 4.

Purpose: telemetry is the "trustworthy at a glance" half of the core value. Output: two
pure snapshot-math helpers, an intraday price helper, an extended header, and sentinel
wiring — all reading the LOADED (pre-write) snapshots.

## Artifacts this plan produces
- state.py: `day_change(snapshots, today) -> float | None` (returns the prior-day baseline value), `n_day_trend(snapshots, today, n=5) -> dict | None` (returns {days, baseline}).
- prices.py: `get_intraday(symbols) -> {sym: {prev_close, last_price}}` (or equivalent), via fast_info.
- notify.py: header telemetry rendering.
</objective>
