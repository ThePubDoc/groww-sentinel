## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Sentinel → Groww API | outbound authenticated calls; credentials cross here |
| Groww API → Sentinel | untrusted external response (holdings/LTP dicts) |
| exception/log strings → operator | error text could carry secret substrings |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-02a | Information Disclosure | Groww token/api_key/seed in logs or on disk | high | mitigate | Token is a local variable, never written to disk (DATA-05); api_key/totp_seed/token never interpolated into exception or log strings (RESEARCH Anti-Pattern). |
| T-01-02b | Spoofing | headless auth to Groww | medium | mitigate | Auth strictly via pyotp-generated TOTP + GrowwAPI.get_access_token; TOTP code never hand-rolled (RESEARCH Don't-Hand-Roll, ASVS V6). |
| T-01-02c | Denial of Service | malformed/partial LTP response | low | mitigate | Missing per-symbol key maps to None (defensive .get), never raises; single batched call stays far under the 300/min live-data quota. |
</threat_model>
