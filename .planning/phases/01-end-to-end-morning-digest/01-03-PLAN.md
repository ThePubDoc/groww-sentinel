<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| env/secrets -> process | four secrets injected; presence validated, values never echoed |
| config.yaml -> process | user-owned local file parsed into the pipeline |
| process -> Telegram API | outbound message; text must carry no secret substrings |
| Groww/Telegram response -> process | untrusted external data (defensive access) |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-03a | Information Disclosure | secret value echoed to stderr/Telegram/logs | high | mitigate | Validate secret presence only; redact any known secret substring from strings headed to stderr/Telegram; never interpolate secret values into messages (RESEARCH Security Domain, ASVS V5). |
| T-01-03b | Tampering | malformed config.yaml -> arbitrary object construction | medium | mitigate | Parse with yaml.safe_load only (never yaml.load with default Loader); bad tag degrades to UNTAGGED, never a hard fail (D-11). |
| T-01-03c | Information Disclosure | secrets committed to git | high | mitigate | .gitignore excludes local .env/secrets file before any commit that could touch credentials; secrets live only in the environment. |
| T-01-03d | Denial of Service | swallowed Telegram/fetch failure -> silent skipped day | high | mitigate | send() calls raise_for_status(); sentinel treats a raised send/fetch error as hard failure (best-effort notify + non-zero exit), never a silent success. |
| T-01-SC | Tampering | pip installs | high | accept | RESEARCH Package Legitimacy Audit verdicts all OK/Approved; no [ASSUMED]/[SUS]/[SLOP] packages -> no legitimacy checkpoint. Versions pinned in requirements.txt (01-01). |
</threat_model>
